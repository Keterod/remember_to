# Remember To.App - Índice de documentación

Documentación inicial del proyecto **Remember To.App**, preparada como contexto técnico para construir la aplicación en Flutter en una etapa posterior.

Esta carpeta no contiene código fuente de la aplicación. Su objetivo es dejar claras las decisiones, conceptos, requisitos y líneas de arquitectura antes de iniciar la implementación.

## Archivos

| Archivo | Descripción |
|---|---|
| `01_vision_alcance.md` | Define la visión del proyecto, problema que resuelve, alcance del MVP, alcance intermedio, alcance avanzado y restricciones. |
| `02_glosario_conceptos.md` | Explica los conceptos principales: actividad, tarea, recordatorio, evento, rutina, tarea mensual y otros términos clave. |
| `03_requisitos_funcionales.md` | Organiza los requisitos funcionales aceptados por módulo, prioridad y versión sugerida. |
| `04_requisitos_no_funcionales.md` | Define requisitos de usabilidad, rendimiento, offline-first, seguridad básica, mantenibilidad, notificaciones y accesibilidad. |
| `05_historias_usuario.md` | Convierte los requisitos principales en historias de usuario con criterios de aceptación. |
| `06_casos_uso.md` | Documenta los casos de uso principales y sus flujos. |
| `07_modelo_datos_inicial.md` | Propone las entidades, campos, relaciones y criterios para SQLite/Drift, incluyendo ocurrencias. |
| `08_arquitectura_tecnica.md` | Propone una arquitectura inicial para Flutter, capas, carpetas, dependencias y flujos técnicos. |
| `09_plan_pruebas.md` | Define un plan práctico de pruebas funcionales, offline, notificaciones, calendario, enlaces, voz y sincronización futura. |
| `10_backlog_sprints.md` | Divide el trabajo futuro en sprints cortos con objetivos, requisitos, tareas y criterios de aceptación. |
| `11_decisiones_tecnicas.md` | Registra decisiones técnicas iniciales para orientar el desarrollo. |
| `12_funciones_descartadas_o_postergadas.md` | Lista funciones descartadas o postergadas y explica por qué no entran en la primera etapa. |
| `13_analisis_apps_similares.md` | Análisis práctico y ligero de aplicaciones similares como referencia de producto. |

## Lectura recomendada

1. Leer primero `01_vision_alcance.md`.
2. Revisar `02_glosario_conceptos.md` para usar el mismo lenguaje durante el desarrollo.
3. Usar `03_requisitos_funcionales.md`, `05_historias_usuario.md` y `06_casos_uso.md` para definir tareas de implementación.
4. Consultar `07_modelo_datos_inicial.md` y `08_arquitectura_tecnica.md` antes de crear el proyecto Flutter.
5. Usar `09_plan_pruebas.md` y `10_backlog_sprints.md` para planificar iteraciones.
